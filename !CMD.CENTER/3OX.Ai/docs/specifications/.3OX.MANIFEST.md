## AUTHORITATIVE STRUCTURE

This manifest defines what **MUST** exist in `.3ox/` and what is **AUTHORITATIVE**.

### Core Identity

## REQUIRED DIRECTORIES

### keys/
**Purpose**: Protected secrets storage
**Linux Equivalent**: `/etc/ssl/private`
**Required**: Yes
**Contents**:
- API keys
- Authentication tokens
- Protected credentials

### vec3/
**Purpose**: Runtime system (programs + data)
**Linux Equivalent**: `/usr` + `/var`
**Required**: Yes
**Structure** (see vec3 manifest):
- `bin/` - Executables
- `lib/` - Libraries
- `rc/` - Run control (service management)
- `var/` - Variable data (runtime state)

## AUTHORITATIVE RULES

1. **.3ox is the desktop** - stays clean and responsive to vec3/
2. **Only core config files** in .3ox root
3. **vec3/ contains runtime** - lib/, rc/, var/
4. **Operations go to** `!CMD.CENTER/!CMD.OPS/`
5. **Planning goes to** `!WORKDESK/3OX.FORGE/plans/`

## FORBIDDEN IN .3OX ROOT

- ‚ùå `pipeline/` - Move to `!CMD.CENTER/!CMD.OPS/Pipeline/`
- ‚ùå `templates/` - Move to `vec3/lib/schemas/` or `!CMD.CENTER/!CMD.OPS/Templates/`
- ‚ùå `rc/` - Duplicate of `vec3/rc/` - use `vec3/rc/` only
- ‚ùå `var/` - Duplicate of `vec3/var/` - use `vec3/var/` only
- ‚ùå Loose Ruby scripts - Move to `vec3/lib/` or `vec3/rc/`
- ‚ùå Documentation files - Move to `!CMD.CENTER/!CMD.OPS/Codex/` or `!CMD.CENTER/!CMD.OPS/Logbook/`

## VEC3 RUNTIME STRUCTURE

### vec3/bin/ (REQUIRED)
**Purpose**: Executable binaries
**Linux Equivalent**: `/usr/bin`
**Required Files**:
- `3ox` or `3ox.exe` - Main executable

### vec3/lib/ (REQUIRED)
**Purpose**: Libraries and modules
**Linux Equivalent**: `/usr/lib`
**Required Subdirectories**:
- `core/` - Core system modules
  - `registry.rb` - Service registry
  - `trace.rb` - Tracing system
- `schemas/` - Data schemas (Z3N.SPEC.md, zen.spec)

### vec3/rc/ (REQUIRED)
**Purpose**: Run control - service management
**Linux Equivalent**: `/etc/init.d` + `/etc/systemd`
**Required Files**:
- `3ox.rc` - Main service controller
**Required Subdirectories**:
- `start.d/` - Start scripts
- `stop.d/` - Stop scripts
- `status.d/` - Status scripts
- `secrets/` - Protected secrets

### vec3/var/ (REQUIRED)
**Purpose**: Variable data - runtime state
**Linux Equivalent**: `/var`
**Required Subdirectories**:
- `state/` - System state
  - `registry.json` - Service registry state
- `log/` - Logs
  - `vec3.trace.log` - System trace log
