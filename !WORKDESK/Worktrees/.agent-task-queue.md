# Agent Task Queue

This file is used for inter-agent task delegation. Agents can assign tasks to each other and update status.

## Task Format
- **ID**: Unique task identifier
- **Assigned To**: Agent name (Agent1, Agent2, Agent3)
- **Assigned By**: Agent who created the task
- **Status**: pending | in_progress | completed | blocked
- **Priority**: low | medium | high | critical
- **Description**: What needs to be done
- **File/Path**: Target file or directory
- **Created**: Timestamp
- **Updated**: Last update timestamp

## Task Delegation Protocol

1. **Creating a Task**: 
   - Add task to "Active Tasks" section
   - Use format: TASK-XXX (sequential numbering)
   - Set status to "pending"

2. **Accepting a Task**:
   - Update status to "in_progress"
   - Update "Updated" timestamp
   - Add "Started" timestamp

3. **Completing a Task**:
   - Update status to "completed"
   - Move task to "Completed Tasks" section
   - Add completion notes

4. **Task Communication**:
   - Agents check this file before starting work
   - Agents update status as they work
   - Use git commits to signal task updates
